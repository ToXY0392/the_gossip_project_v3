# Documentation des améliorations – The Gossip Project

Ce dossier contient les améliorations proposées pour le projet, **classées par ordre de priorité**.

## Fichiers

| Fichier | Priorité | Contenu |
|--------|----------|--------|
| [01-priorite-haute.md](01-priorite-haute.md) | Haute (court terme) | 404, tests, N+1, validations modèles |
| [02-priorite-moyenne.md](02-priorite-moyenne.md) | Moyenne (moyen terme) | Auth, email, welcome, tags, commentaires, routes |
| [03-priorite-basse.md](03-priorite-basse.md) | Basse (plus tard) | CRUD gossips, likes, MP, pagination, typo, SEO, i18n |

## Résumé

- **Priorité haute** : robustesse (404, tests, N+1) et base de données (validations).
- **Priorité moyenne** : sécurité et UX (authentification, données sensibles, tags, commentaires, routes).
- **Priorité basse** : fonctionnalités avancées (CRUD, likes, messages privés, pagination) et polish (typos, SEO, i18n).

Travailler dans cet ordre permet de stabiliser l’app puis d’ajouter les fonctionnalités métier.
